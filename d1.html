github在线展示前端项目页面

最近在慢慢构建自己的主页，想把之前做的一些可视分析系统放在主要上展示，但是要生成ip地址比较麻烦，对比来看，使用github.io的功能算是比较简单易维护的。在网上看了几个教程，把比较有用的放在这里，以便快速查询。

  1. git维护常用代码

    2. 提示：21年7月后已经不能用账号加密码登入，必须要用personal token，查看非常有用的链接3。

    1. 创建空目录

    
mkdir (repository_name) ## 在合适路径里创建空目录
cd (repository_name) 
pwd ##显示当前目录

    2. 把目录变成git仓库 git init。如果是本地已经有的project，进入project所在目录，从这一步开始，上传所有文件用git add .注意此处add后有一个dot

    3. 修改readme.txt文件放到仓库

    git add readme.txt
git commit -m "wrote a readme file"  ## 本次提交的说明

    注意：可以一次add多个，一起commit；add其实是把修改记录到了暂存区，而不是把文件记录到了暂存区，所以add之后再对文件进行的修改不会在commit使被commit，必须要在修改之后再add一次。也就是说commit只提交已经被存到暂存区的修改。

    4. 远程上传到暂存区

    git remote add origin https://github.com/xxxxxx/xxxxxxxxxxxxxxxx

    5. 远程上传到github仓库

    git push -u origin master

    6. 其他操作：a.查看仓库当前状态git status. b.查看没被commit的修改

    git dff. c.查看历史修改记录git log ；回退上个版本git reset --hard HEAD^。注意：当前版本用HEAD表示，上一个版本用HEAD^表示，上上一个版本用HEAD^^表示，前100个就是HEAD~100 如果要返回到新的版本，可以找新的版本的commit id，使用git reset --hard xxxxx写前几位即可；可以使用git reflog输出历史命令找回commit id

    d.删除文件并删除版本库内的文件git rm test.txtgit commit -m "remove test.txt" e.误删文件从版本库恢复git checkout -- test.txt
