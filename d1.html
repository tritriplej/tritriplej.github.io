关于git和github的记载

最近在慢慢构建自己的主页，想把之前做的一些可视分析系统放在主要上展示，但是要生成ip地址比较麻烦，对比来看，使用github.io的功能算是比较简单易维护的。过程中在网上看了几个教程，把比较有用的放在这里，以便快速查询。

  1. git的日常使用

    2. 重要变动：21年7月后在本地登陆时使用的密码为personal token，并且需要将remote URLs从SSH变为HTTPS。查看「非常有用的链接」。

    在本地维护远程仓库有Github Desktop和使用git bash终端两种方法，用Github Desktop比较用户友好，使用时不容易产生问题，这里主要记录的是终端维护的常用代码和问题，初始的安装下载配置config问题见「非常有用的链接」git教程。

    git操控远程仓库常用代码
    ## 进入本地project文件路径
git init ## 把目录变成git仓库，只有第一次要做
ls -ah ## 可以看到多了一个.git文件夹
git add .  ##注意此处add后有一个dot 如果文件没有变动则没东西
git add readme.txt  ##修改readme.txt文件放到暂存区
git commit -m "wrote a readme file" ## 提交修改的说明
git remote add origin https://github.com/username/repositoryname.git ## 远程上传到暂存区
git push -u origin master ## master 是当前origin的branch名字 ## 远程上传到github仓库
    其他常用操作

    注意：当前版本用HEAD表示，上一个版本用HEAD^表示，上上一个版本用HEAD^^表示，前100个就是HEAD~100。

    ## 进入本地project文件路径
git status ## 查看仓库当前状态
git diff ## 查看没被commit的修改
git log ##查看历史修改记录
git reset --hard HEAD^  ##回退上个版本

##退回到特定版本
git reflog ## 输出历史命令找回commit id
git reset --hard xxxxxx ##填入commit_id

## 删除文件并删除版本库内的文件
git rm test.txt
git commit -m "remove test.txt"

## 误删文件从版本库恢复
git checkout -- test.txt
